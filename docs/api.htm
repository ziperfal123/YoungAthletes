Exercise 1 - Young Athletes

Comments (0)

A Web service that handles a list of young athletes.

View project on GitHub: https://github.com/ziperfal123/YoungAthletes

API

This part will cover:

  • Requests for fetching / updating documents from the DB Collection
  • Responses to the client

GET Fetch all Athletes from DB

https://young-athletes.herokuapp.com/getAllAthletes

This requets will fetch all of the documents stored in the DB collection. The result will appear in the client browser as a JSON object.

Example RequestGET - Fetch all Athletes from DB
curl --location --request GET "https://young-athletes.herokuapp.com/getAllAthletes"
Example Response200 OK
[

            "sport_name": "Swimming",

            "best_record": 21

        "first_name": "reba",

GET Fetch athletes using a filter

https://young-athletes.herokuapp.com/getAthletesBySportNameBestRecord?sportName=running&bestRecord=31

This request has to get 2 parameters via the queryString:

  1. sportName - The sport type. e.g Swimming, Running etc. NOTICE: all letters has to be small.
  2. bestRecord - A least upper bound of the best records.

The athletes that will be fetched from the DB will be the ones that are from the relevant sport type (the one that was given in the first parameter), and their best record is better (=lower) then the record that was sent as a parameter.

Params

sportNamerunning
bestRecord31
Example RequestFetch athletes using a filter
curl --location --request GET "https://young-athletes.herokuapp.com/getAthletesBySportNameBestRecord?sportName=running&bestRecord=31"

POST Update best record

https://young-athletes.herokuapp.com/updateRecord

This request handles update for a specific document in the DB collection. 2 Parameters are being sent via the body message:

  1. id - The athlete id

  2. bestRecordToUpdate - a new, numeric (with/without floating point) best record to update.

    The new 'best record' has to be better then the one that the athlete has in the same moment. If his/hthe athlete's old record is better then the new one, the update won't happen. If the new record is better, the athlete data will be updated accordingly.

Headers

Content-Typeapplication/x-www-form-urlencoded

Body

urlencoded
id1928675849
bestRecordToUpdate19.22
Example Request
Update best record
curl --location --request POST "https://young-athletes.herokuapp.com/updateRecord" \
  --header "Content-Type: application/x-www-form-urlencoded" \
  --data "id=1928675849&bestRecordToUpdate=19.22"
Example Response200 OK
"The athlete with the id number: '1928675849' was Updated succesfully"

Error handling

This part will cover:

  • Invalid route errors.
  • Invalid record validation - A case that the client sends an invalid record (e.g non-numeric characters)

GET Invalid route example

https://young-athletes.herokuapp.com/someWrongRoute

An example of an answer that is being sent from the server when a user is entering an invalid route.

Example RequestInvalid route example
curl --location --request GET "https://young-athletes.herokuapp.com/someWrongRoute"
Example Response200 OK

POST Invalid record example

https://young-athletes.herokuapp.com/updateRecord

An example for an answer that will be sent to the client when sending an invalid record (e.g non-numeric record) as parameter (notice the latter 'd' in the 'bestRecordToUpdate' input parameter)
